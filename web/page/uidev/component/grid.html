关于容器

容器（container）是bootstrap框架定义的一个布局类，一般与栅格系统配套使用。该类有以下特点

（1）默认带有居中以及padding样式，因此无法嵌套使用。
（2）根据屏幕大小，会自动调整其宽度，这个大小为：

超小屏幕设备 手机 (<768px) 小屏幕设备 平板 (≥768px) 中等屏幕设备 桌面 (≥992px) 大屏幕设备 桌面 (≥1200px)
自适应 750px 970px 1170px

使用以下代码，并调整窗口宽度可以看到这个效果：

		$(window).resize(function(e){
			var o = $("#testdiv_1");
			var w = o.css("width");
			o.html("我的宽度为：" + w);
		});
此处改变窗口大小后会显示容器大小

但是在有些时候，不需要容器具有居中效果，这时就要去掉容器的margin-left样式。

		<div class="container" id="testdiv_2" style="margin-left:8px;background-color:#EEEEEE">
			设置固定margin-left的容器
		</div>

什么是栅格系统

栅格系统是一系列CSS组成的响应式布局，该布局会根据屏幕大小的改变，而表现出不同的样式。栅格系统有以下特点：

（1）栅格系统通过行和列划分区域，一般来说，行需要放置在容器中，但是这个容器也可以是表单域（form）。
（2）列应该放置在行中，用户创建的元素放置在列中，列与列之间有默认的间距。
（3）栅格系统将一行划分为12列，每个元素可以通过CSS占据连续的多个等宽列，也可以通过设置margin来产生位移，或者弥补padding。
（4）栅格系统定义大量形如.col-xs-4的样式，其中xs表示非常小的屏幕，如移动设备，4表示占据连续4列。

栅格系统的特点以及CSS前缀如下：

超小屏幕设备 手机 (<768px) 小屏幕设备 平板 (≥768px) 中等屏幕设备 桌面 (≥992px) 大屏幕设备 桌面 (≥1200px)
栅格系统行为 总是水平排列 开始是堆叠在一起的，超过这些阈值将变为水平排列
最大容器宽度 None (自动) 750px 970px 1170px
class前缀 .col-xs- .col-sm- .col-md- .col-lg-
列数 12
最大列宽 自动 60px 78px 95px
槽宽 30px (每列左右均有15px)
可嵌套 Yes
偏移（Offsets） Yes
列排序 Yes

这里特别强调的是，从代码的角度来看，列宽的真正定义是百分比，而不是绝对宽度，比如以下代码定义了在小屏幕下可以并排显示的三个红色区域，但是因为没有容器的包裹，这三个红色区域连起来占据了整个屏幕看度，但是又可以根据表中的阈值调整其布局。

利用栅格系统创建自适应的表单

在栅格系统中，假如设置了md屏幕大小的样式，那么对于lg大小的屏幕，该样式一样适用，但是反过来，如果屏幕小于md时，根据栅格系统的设定，这些元素就会变为堆叠布局。因此，为了制作自适应的表单，需要为一个元素设置至少2个样式。

下面的代码使用栅格系统创建了一个在大屏幕上显示3列，而小屏幕下显示2列的表单（在“输入框”一节中同样是该代码）：

		<form class="form-horizontal bbox" role="form">
			<div class="row">
				
				<div class="form-group col-xs-6 col-md-4">
					
					<label class="col-xs-4 control-label">姓名
					<div class="col-xs-8">
						<input type="text" class="form-control">
					</div>
				</div>
				
			</div>
		</form>

如果需要自动布局的列多余3，请都写在一个row中，因为row有换行的效果。对于某些系统而言，这个代码还有问题，首先我们需要标签大小尽可能固定，否则在大屏幕下左侧会有较大空隙，此时，列大小就不能使用列样式控制，而需要设定为固定值：

		<form class="form-horizontal bbox" role="form">
			<div class="row">
				<div class="form-group col-xs-6 col-md-4">
					<table width="100%">
						<tr>
							<td width="120px" class="ctrl-label">
								姓名：
							</td>
							<td>
								<div class="input-group-sm">
									<input type="text" class="form-control">
								</div>
							</td>
						</tr>
					</table>
				</div>
				<!--剩下的写法一致-->
			</div>
		</form>

和上面的代码不同，该代码做了以下修改：

（1）虽然列和列之间有默认的间距，但是此段代码的“label”和输入框是在一起的，因而需要调整好label的宽度；
（2）在不使用JS的情况下，无法设置“百分比-定宽”样式的CSS，但是表格支持；
（3）label在表格中，失去其原有样式，所以这里直接设置右对齐反而更简单一些；

该代码依然实现了3列/2列的布局，而且文本框的长度会自动填满列宽。

列偏移

为元素添加col-XX-offset-X形式的样式可以做到列偏移（左边距）的效果。

		<row>
			<div class="col-xs-6 col-xs-offset-1">
				offset=1/12
			</div>
			<div class="col-xs-6 col-xs-offset-2">
				offset=2/12
			</div>
			<div class="col-xs-6 col-xs-offset-3">
				offset=3/12
			</div>
			<div class="col-xs-6 col-xs-offset-4">
				offset=4/12
			</div>
		</row>

响应式布局

bootstrap定义了以下样式，可以用于在特定屏幕下显示/隐藏某些内容。

超小屏幕 手机 (<768px) 小屏幕 平板 (≥768px) 中等屏幕 桌面 (≥992px) 大屏幕 桌面 (≥1200px)
.visible-xs 可见 隐藏 隐藏 隐藏
.visible-sm 隐藏 可见 隐藏 隐藏
.visible-md 隐藏 隐藏 可见 隐藏
.visible-lg 隐藏 隐藏 隐藏 可见
.hidden-xs 隐藏 可见 可见 可见
.hidden-sm 可见 隐藏 可见 可见
.hidden-md 可见 可见 隐藏 可见
.hidden-lg 可见 可见 可见 隐藏

需要注意的是，这里的visible和hideden不具有叠加效果，如下面的代码：

		  <div class="visible-xs visible-sm visible-md visible-lg row">这个DIV会在超小屏幕或者以上显示</div>
	      <div class="visible-sm visible-md visible-lg row">这个DIV会在小屏幕或者以上显示</div>
	      <div class="visible-md visible-lg  row">这个DIV会在中等屏幕或者以上显示</div>
	      <div class="visible-lg row">这个DIV会在大屏幕或者以上显示</div>
